# Sec-Agent-Harness: 技术设计与实现报告 (中文版)

## 1. 项目概述
本项目旨在打造一个底层基建级别的 Agent 框架，专门解决 LLM 在长周期安全任务中“中途迷失”和“性能损耗”的问题。核心特性包括：
- **混合运行时**：兼顾极致性能与生态兼容。
- **强控状态机 (FSM)**：确保漏洞挖掘路径的可预测性。
- **上下文压实**：大幅降低 Token 消耗。

---

## 2. 核心设计哲学

### 2.1 混合运行时 (Hybrid Runtime)
- **内核态 Skill (Kernel-mode)**：对于 50MB 以上的 AST 深度遍历等计算密集型任务，直接在 Agent 进程空间执行。**优势**：零拷贝、零序列化延迟、保留对象内存引用。
- **协议化扩展 (MCP)**：对于 Slack 告警、Jira 工单、GitHub API 等外围操作，采用标准 MCP 协议。**优势**：解耦、拥抱开源生态。

### 2.2 防御性沙箱 (Defensive Sandbox)
- 遵循“最小权限原则”，所有 LLM 生成的代码（PoC/Fix/Test）均在 Docker 或隔离容器内执行。
- **资源熔断**：设置 cgroups 限制，防止 OOM 或 Fork 炸弹拖垮宿主机。

### 2.3 上下文压实与指针机制
- 借鉴 `Persisted Output Marker` 机制，Agent 间交互只传递轻量级的“特征摘要指针”，而非庞大的原始日志。模型按需读取详情，Token 消耗可降低约 60%。

---

## 3. 架构组件说明

### 3.1 核心引擎 (`core/`)
- **`loop.py`**：实现 `Request -> Response -> Tool -> Result` 的循环，是系统的“心脏”。
- **`skill.py`**：技能注册中心。采用“目录摘要注入 + 按需加载详情”的双层模式，保证系统提示词（System Prompt）的精简。

### 3.2 安全技能库 (`skills/`)
- **污点分析 (Taint Analysis)**：引导 LLM 进行结构化推理，自动映射 Source 到 Sink 的路径。
- **漏洞验证 (Validator)**：(规划中) 利用约束求解器过滤误报。
