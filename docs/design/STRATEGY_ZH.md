# Sec-Agent-Harness 开发策略 (中文版)

## 愿景 (Vision)
构建一个高性能、专门面向复杂逻辑漏洞挖掘与自动修复的 Agent 系统。它将结合 "Claude Code" 的灵活 Harness 架构与专业的安全逻辑（如污点分析、约束求解、隔离沙箱）。

## 实现阶段 (迭代式)

### 第一阶段：基础设施 (核心循环与 Skill 系统)
- **目标**：实现类似 `learn-claude-code` 中 `s01-s05` 的核心功能。
- **组件**：
    - **最小化循环 (Minimal Loop)**：一个健壮的“消息-工具-结果”闭环。
    - **内核态 Skill 注册中心**：一套能够在进程内加载高性能安全工具（如 AST 解析器、CFG 构建器）的机制，避免 RPC 序列化开销。
    - **基础上下文管理**：初步实现消息剪枝与压实（Compaction）。

### 第二阶段：安全专业化 (沙箱与污点分析)
- **目标**：引入防御性执行环境与符号化推理能力。
- **组件**：
    - **防御性沙箱 (Defensive Sandbox)**：基于 Docker 或 E2B 的隔离环境，用于运行生成的 PoC 或测试用例。
    - **污点驱动推理**：通过结构化 JSON 规则引导大模型进行数据流追踪的“Skill”。

### 第三阶段：多智能体与状态机 (最终重构目标)
- **目标**：编排多个专业智能体协同工作。
- **组件**：
    - **FSM 编排器 (FSM Orchestrator)**：在“静态分析”、“漏洞验证”、“修复 (Fixer)”和“审计 (Reviewer)”状态间自动流转。
    - **基于指针的上下文压实**：使用 `Persisted Output Markers` 在 Agent 间传递重型分析结果，而不占用 Token 空间。

## 架构融合裁决 (逻辑链条)

经过对三个技术方向（FSM 强控编排、神经符号引擎、动态反馈沙箱）的深度探索，我们认为单一范式无法满足工业级自动化漏洞修复的需求，因此确立了 **“大一统融合架构” (Unified Sec-Agent Harness)**：

1. **FSM 骨架 (控制与韧性)：** 纯大模型在长周期任务中容易迷失。我们采用结构化的有限状态机（初始分析 -> 验证器 -> 修复器 -> 审查员），强制执行确定性的状态流转与熔断回溯，保证系统不陷入死循环。
2. **神经符号节点 (精准定位)：** 在“初始分析”阶段，纯大模型容易产生幻觉。我们注入“内核态 AST 解析技能”，让 LLM 输出结构化 JSON 规则，交由底层的静态引擎和 Z3 约束求解器执行污点追踪。这能在数学层面过滤误报，保证移交给下游的漏洞质量。
3. **动态沙箱节点 (实战反馈)：** 在“验证”与“审查”阶段，静态分析无法证明漏洞的真实可利用性，也无法保证修复不破坏业务逻辑。我们将 PoC 和补丁放入 Docker/eBPF 隔离沙箱中执行。通过捕获真实的报错和内存快照，LLM 可以演化式地变异 PoC；同时，修复补丁必须通过严格的回归测试（安全与功能的双重约束），才能被最终采纳。

**逻辑链条总结：** FSM 提供**整体的控制边界** -> 神经符号协同提供**高精度的静态靶点** -> 动态隔离沙箱提供**基于真实执行的闭环反馈**。
