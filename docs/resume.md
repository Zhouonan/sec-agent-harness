**Sec-Agent-Harness — 基于状态图编排的安全审计 Agent 系统**

项目背景：基于已有审计项目进一步设计的 Agent 协同系统，借鉴 LangGraph StateGraph 与 Reducer 归约思想，通过确定性工作流架构解决 LLM 在长链路安全任务中的不确定性问题，实现漏洞发现到自动修复的端到端闭环。

**确定性多阶段工作流编排**：将审计流程拆解为四阶段功能节点（分析→验证→修复→审查），核心控制策略包括：①确定性条件路由——节点流转由代码基于 State 中客观结果（测试退出码等）强制驱动，而非依赖 LLM 自主决策；②节点级工具权限隔离——每个节点绑定独立 System Prompt 与专属工具集，执行最小权限原则；③子图嵌套——将需要多轮自主纠错的阶段（如 VALIDATOR）封装为内嵌 ReAct 循环的独立子图，对主流程透明；④Human-in-the-Loop 断点——在高危节点前设置静态中断，支持人工审批后恢复执行。

**State 增量通信与 Reducer 归约**：节点仅返回增量字典，由引擎通过 Reducer 统一合并至全局 State，保证变更可溯源；State 内结构化数据（执行计划、阶段摘要）以 JSON 格式注入 Prompt 独立区块，提升模型对任务进度的可观测性；配合节点级 Checkpoint 落盘，支持崩溃恢复与 Time Travel 状态回放调试。

**分层上下文生命周期管理**：设计工作记忆与持久记忆分离策略——状态切换时强制执行消息摘要压缩，关键结论沉淀至 State 持久层跨节点保留；实现 State 膨胀检测与自动归档，超阈值时自动清理历史摘要与过期 Skill 缓存；大体量数据（编译日志、扫描报告）外置为按需检索资源，避免上下文窗口污染。

**声明式 Skill 插件体系**：将静态分析能力（AST 全局调用图构建、跨文件污点传播追踪）封装为独立 Skill 插件，通过 YAML Manifest 声明工具接口与作用域，引擎启动时自动发现注册；Skill 以内核态零序列化开销直接向 LLM 提供数据流拓扑，弥补模型在大规模代码仓库上的全局视野盲区。

## v4（实战问题驱动 + 自主设计视角）

**Sec-Agent-Harness — 面向安全审计的自主 Agent 系统**

**FSM 工作流引擎（手搓实现）**：自研有限状态机调度核心，核心解决三个实战问题：①模型"忘记切换状态"——设计确定性条件路由，基于 State 中客观结果（测试退出码、黑板标记）由代码强制驱动阶段流转，彻底剥离 LLM 对控制流的决策权；②工具越权调用——实现节点级工具权限绑定（`Dict[AgentState, List[Tool]]`），每个阶段仅暴露该职责范围内的工具集，如沙箱执行仅对 VALIDATOR/REVIEWER 可见；③工具调用死循环——设计双层熔断机制（全局轮次上限 + 单状态轮次上限），配合空调用自动注入 Nudge 系统消息驱动模型恢复行动。

**结构化 State 与上下文压缩策略**：以 `LoopState` 为全局共享状态，通过 Blackboard 字典承载跨节点持久化数据（分析线索、测试状态、阶段摘要），并以 JSON 格式动态注入 System Prompt 独立区块，提升模型对任务进度的可观测性；针对长对话 Token 爆炸问题，设计三级压缩机制：状态切换时强制清理工作记忆仅保留原始任务、State 膨胀超阈值时自动归档非当前阶段摘要并淘汰过期 Skill 缓存、大体量工具输出（编译日志等）执行首尾截断保留关键信息。

**声明式 Skill 插件与动态 Prompt 组装**：将静态分析能力（AST 调用图构建、跨文件污点传播）封装为独立 Skill 插件，通过 YAML Manifest 声明工具接口，引擎启动时自动扫描注册；Agent 运行时通过 `load_skill` 工具按需加载 Skill 内容至 State，由引擎在每轮 Prompt 构建时动态组装 System Prompt（角色指令 + Skill 目录 + 执行计划 + Blackboard 上下文），实现 Prompt 内容随阶段和上下文自适应变化。
