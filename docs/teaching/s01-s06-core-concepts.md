# 核心原理教学：从基础循环到上下文治理 (s01 - s06)

本指南旨在帮助你通过 `Sec-Agent-Harness` 的实现，理解 `learn-claude-code` 中最核心的 6 个阶段。我们将对比原仓库的“通用型设计”与我们“安全增强型设计”的区别。

---

## 1. 基础心跳：Agent 循环与工具调用 (s01 - s02)

### 原理简述
- **s01 (Loop)**: Agent 的本质是一个 `while` 循环。它不断问 LLM：“下一步做什么？”，如果 LLM 返回“我要调工具”，Agent 就去执行工具并将结果喂回给 LLM。
- **s02 (Tool Use)**: 引入了结构化的工具定义 (JSON Schema) 和分发器 (Dispatcher)。

### 我们的实现 vs 原仓库
- **原仓库**: 采用扁平化的函数式循环。
- **Sec-Agent (改进)**: 我们采用了 **类对象化管理 (`AgentLoop` 类)**。
    - **改进点**: 我们在 `core/loop.py` 中预留了 `AgentState` 枚举。这是为了防止安全任务中常见的“工具死循环”——如果 LLM 在 `INITIAL_ANALYSIS` 阶段卡住了，我们可以在类成员中轻松记录并触发熔断。

---

## 2. 规划与隔离：从 Todo 到状态机 (s03 - s04)

### 原理简述
- **s03 (Planning)**: 给 Agent 一个“笔记本”记录待办事项，防止它迷失方向。
- **s04 (Subagent)**: 遇到复杂任务时，开一个“小号”去干活，干完只把结果拿回来，不污染主对话的上下文。

### 我们的实现 vs 原仓库
- **原仓库**: 侧重于显式的 `todo.txt` 文件和临时的子智能体调用。
- **Sec-Agent (重大改进)**: 我们直接将“隔离”提升到了架构级，引入了 **有限状态机 (FSM)**。
    - **逻辑链条**: 在安全审计中，`分析` 和 `验证` 的逻辑是完全不同的。我们不只是开子智能体，而是直接切换 **状态 (AgentState)**。每个状态有独立的 `System Prompt` 和 `Tool Set`。这比原仓库的 Subagent 更重、更稳，确保“分析员”不会误操作去执行高危的“验证”代码。

---

## 3. 技能按需加载：内核态 Skill (s05)

### 原理简述
- **s05 (Skills)**: 不要把所有说明书都塞给 LLM。先给它看一个“目录”，它想用哪个技能时，再通过 `load_skill` 加载详细说明。

### 我们的实现 vs 原仓库
- **原仓库**: 动态加载 Markdown 文件。
- **Sec-Agent (改进)**: 我们引入了 **“内核态 (Kernel-mode)”** 的概念。
    - **改进点**: 在 `core/skill.py` 中，我们的注册中心不仅能加载文档，还设计为能加载 **Python 原生模块**。
    - **为什么要改？** 在处理 50MB 的 AST 解析时，原仓库的“文本传递”会直接导致系统崩溃。我们的设计允许 LLM 像调用内核驱动一样调用高性能的本地分析器，只返回“结果指针”。

---

## 4. 瘦身艺术：上下文压实与黑板机制 (s06)

### 原理简述
- **s06 (Context Compaction)**: 对话太长了，LLM 会变笨且变贵。需要总结过去的对话，删掉无用信息。

### 我们的实现 vs 原仓库
- **原仓库**: 定期进行文本总结 (Summarization)。
- **Sec-Agent (重大改进)**: 我们引入了 **黑板机制 (Blackboard)**。
    - **核心区别**: 我们不只是总结文本。在 `core/loop.py` 中，`blackboard` 是一个持久化的字典。
    - **逻辑**: 状态跳转（如从 `分析` 到 `验证`）时，我们强行清空 `messages` 历史（除了最核心的指令），只保留 `blackboard` 里的结构化数据（如漏洞的路径、变量名）。
    - **效果**: 这比简单的文本总结更精准，能让 Token 消耗降低 60% 以上，同时保证“验证状态”的智能体拥有完美的“信噪比”。

---

## 5. 总结：给学习者的建议

如果你想深度掌握这部分，请重点查看 `core/loop.py` 中的 `transition_handler` 函数。它完美融合了 s04 (隔离)、s03 (规划) 和 s06 (压实) 的思想：
1. 它完成了状态跳转（规划）。
2. 它清理了冗余上下文（隔离）。
3. 它将关键信息沉淀到了黑板上（压实）。
